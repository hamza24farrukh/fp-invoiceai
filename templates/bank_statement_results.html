How Matching Works

The processed bank statement includes the following enriched columns:

Column Description
Payee Supplier/recipient identified by matching against your invoice database
Category Service category from the matched supplier record
Transaction Type Transaction classification from the matched invoice
matched_supplier The supplier name that was matched, or empty if no match was found
match_confidence 0-100% score indicating how confident the algorithm is about the match
converted_invoice_amount The invoice amount converted to the bank statement's currency for comparison
amount_difference_pct Percentage difference between converted invoice amount and bank transaction (accounts for transfer fees)
match_details Human-readable breakdown showing the currency conversion and comparison
Matching Algorithm

Each transaction is scored against all invoices using four weighted criteria:

  • Amount Closeness (55%) — converts invoice amount to bank currency, then compares. Allows up to 15% deduction for SWIFT fees, intermediary charges, and local taxes on international transfers
  • Date Proximity (20%) — how close the transaction date is to the invoice date (up to 30 day window for international transfers)
  • Supplier Name (15%) — checks if the supplier name appears in the bank transaction description
  • Invoice Number (10%) — checks if the invoice number appears in the transaction reference

Matches require a minimum 30% confidence score. Credit transactions are matched against income invoices, debit transactions against expense invoices.

Tip: Unmatched transactions may indicate missing invoices in your system. Upload the corresponding invoices first, then re-process the bank statement for better match rates.
